# Іконки маршрутів (підготовка)

Іконки маршрутів лежать у `resources/icons/tracks/routes/{Назва карти} - {Назва треку}.png`
і показуються в списку треків (вкладка Tracks). Назва файлу має точно збігатися з парою
`map_name` + `name` треку в `services/initial_data.py` — за нею іконка підтягується в БД
(`initial_data.sync_track_icons`, колонка `tracks.icon`). Префікс карти — лише для зручності
орієнтування у файлах.

## Звідки беруться

Вручну — скріншоти екрана вибору маршруту з гри. На сирому скріншоті темна картка з
треком лежить на білому тлі, під нею білий футер із назвою, а в правому нижньому куті —
індикатор тривалості («30''» з дужкою). Назву треку читаємо з футера й називаємо файл як
`{Назва карти} - {Назва треку}.png`.

## Підготовка

`scripts/crop_route_icons.py` проганяє кожен файл через конвеєр:

1. **Обрізка картки** — найбільша зв'язна non-white область; прибирає білі поля й футер із
   назвою. Білі лінії самого маршруту всередині картки зберігаються (вони з'єднані з фоном).
2. **Видалення індикатора тривалості** — дужка з «30''» у правому нижньому куті заливається
   кольором фону (фон рівний, тож безшовно). Дужка унікально сіра, тож відрізняється від
   фіолетового/білого маршруту.
3. **Центрування на квадраті 1:1** — контент (маршрут) центрується на квадратному полотні з
   невеликими полями.

Скрипт **ідемпотентний** — повторний запуск на вже обробленому файлі дає піксельно той самий
результат (можна безпечно ганяти на всій теці).

```powershell
# обробити всі іконки на місці (типово resources/icons/tracks/routes)
.\.venv\Scripts\python.exe -m scripts.crop_route_icons

# окремі файли
.\.venv\Scripts\python.exe -m scripts.crop_route_icons "resources/icons/tracks/routes/The Caribbean - Hell Vale.png"

# прев'ю в окрему теку, не чіпаючи оригінали
.\.venv\Scripts\python.exe -m scripts.crop_route_icons --out _preview

# лише показати, що буде зроблено
.\.venv\Scripts\python.exe -m scripts.crop_route_icons --dry-run
```

Опції: `--out DIR`, `--margin FRAC` (поле навколо контенту, типово 0.05), `--dry-run`.

### Перекриття індикатора маршрутом

Якщо лінія маршруту заходить у прямокутник індикатора, безпечно прибрати його не можна —
скрипт **пропускає такий файл незмінним**, виводить його у списку «на ручну обробку» і
завершується з кодом виходу 1. Такі 1–2 треки доводиться чистити вручну (стерти індикатор у
редакторі), після чого повторний прогін скрипта довідцентрує й зробить квадрат.

## Превʼю для списку

Повна іконка обрізана по всьому контенту картки (біла лінія маршруту + фіолетова «місцевість»).
Біла лінія часто витягнута по ширині, тож на квадратній мініатюрі списку виглядає дрібно. Тому
поряд тримаємо **превʼю** — квадратний кроп оригіналу навколо самої білої лінії з невеликим
паддингом (зображення не змінюється, у паддинг можуть потрапити сусідні фіолетові дороги — це ок).

Превʼю лежать у `resources/icons/tracks/routes_preview/{та сама назва}.png`, синхронізуються в БД
в окрему колонку `tracks.icon_preview` (`initial_data._bundled_track_preview` /
`sync_track_icons`) і показуються як мініатюра в списку треків; tooltip при наведенні лишається
на повній іконці.

```powershell
# згенерувати превʼю для всієї теки routes -> routes_preview
.\.venv\Scripts\python.exe -m scripts.crop_route_preview

# окремі файли / інша тека виводу / показати без запису
.\.venv\Scripts\python.exe -m scripts.crop_route_preview "resources/icons/tracks/routes/Paris - Metro.png"
.\.venv\Scripts\python.exe -m scripts.crop_route_preview --out _preview
.\.venv\Scripts\python.exe -m scripts.crop_route_preview --dry-run
```

Опції: `--out DIR` (типово `routes_preview`), `--margin FRAC` (паддинг, типово 0.08), `--dry-run`.
Файл без білих пікселів пропускається (превʼю нема → список покаже повну іконку).

## Процедура додавання нового маршруту

1. Поклади сирий скріншот у `resources/icons/tracks/routes/` і назви як
   `{Назва карти} - {Назва треку}.png` (точно як пара `map_name`/`name` у `initial_data.py`),
   напр. `Paris - Metro.png`.
2. Прожени: `.\.venv\Scripts\python.exe -m scripts.crop_route_icons` (безпечно для всієї теки —
   готові файли не змінюються).
3. Якщо файл потрапив у «на ручну обробку» — прибери індикатор у редакторі й прожени скрипт ще
   раз (тепер він довідцентрує й зробить квадрат).
4. Згенеруй превʼю: `.\.venv\Scripts\python.exe -m scripts.crop_route_preview`.
5. Перезапусти застосунок — `sync_track_icons` підхопить нову іконку та превʼю для треків без
   власних (правки користувача, зроблені через діалог Edit Track, не перезаписуються).

> Наразі без іконок лишаються треки без скріншота: **A Run in the Park**,
> **Friendly Neighborhood**, **Metro**, **Islet Race**, **Versatile Trail**.
