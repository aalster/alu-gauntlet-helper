# Capture: інлайн-рев'ю в табі замість модалки

Дата: 2026-06-12

## Мета

Прибрати `CaptureReviewDialog`. Весь функціонал рев'ю захоплених гонок переїжджає
прямо в `CaptureTab`: кожна гонка має чекбокс і кнопку редагування (відкриває той
самий `RaceDialog`, що й у races-табі), одна кнопка сабміту зберігає всі вибрані гонки.

## Ключові рішення (з обговорення)

1. **Ручні правки фіксують поля пофілдово.** Заповнене руками поле OCR не
   перезаписує. Якщо юзер стер значення поля — воно знову відкрите для OCR.
2. **Мініатюри скріншотів (`panel_image`) в UI не показуються ніде.**
   (Саме поле `RaceCapture.panel_image` і його захоплення не чіпаємо.)
3. **Чекбокс на кожній гонці.** Сабміт зберігає тільки чекнуті. За замовчуванням
   чекбокс увімкнений у гонок із заповненим часом (гравець може не грати всі 5).
4. **Непевний OCR** (score < 0.85): перед значенням поля — жовта іконка «⚠».
5. **Архітектура: шар драфтів у `ChallengeSessionService`** (підхід 1).
   `bad_timing` і `note` входять у драфт (юзер відмічає аварію як причину
   поганого часу).
6. **Оверлей показує ті самі ефективні дані, що й таб** (трек/авто/час);
   `bad_timing`/`note` в оверлеї не потрібні.

## Дизайн

### 1. `ChallengeSessionService` — шар драфтів

- Нове поле `drafts: dict[int, RaceView]` — ручні правки по номеру гонки.
  Поля драфта можуть бути «порожніми» (`track_id=0`, `car_id=0`, `time=0`,
  `rank=0`, порожній `note`) — порожнє поле «просвічує» OCR-шар.
- `set_draft(race_number: int, draft: RaceView)` — зберігає драфт, викликає `_notify()`.
- `effective(race_number) -> EffectiveRace | None` — зливає драфт поверх OCR:
  - значення поля = непорожнє значення з драфта, інакше — з OCR;
  - `EffectiveRace` (датаклас/модель у `challenge_session.py`): `track_id`,
    `car_id`, `rank`, `time`, `bad_timing`, `note`, `track_uncertain`,
    `car_uncertain`;
  - `track_uncertain`/`car_uncertain` = true, якщо значення взяте з OCR зі
    score < 0.85; для перекритих драфтом полів — завжди false;
  - повертає `None`, якщо по гонці немає ні OCR-захоплення, ні драфта.
- `clear()` скидає і `races`, і `drafts`.
- OCR-мердж (`apply`/`_merge`) не змінюється — пише тільки у свій шар.

### 2. `RaceDialog` — режим послабленої валідації

- Параметр `relaxed: bool = False` у конструкторі.
- При `relaxed=True` `prepare_item` не вимагає трек/авто/час: порожні поля
  повертаються як «не задано» (track_id=0, car_id=0, time=0) без `set_error`.
  Юзер стер поле → OCR зможе дозаповнити його пізніше.
- Заголовок у capture-режимі: «Edit Race N».

### 3. `CaptureTab` — новий вміст

Кожен із 5 рядків:

```
[✓] Race 1   Nest - Sea Walk · ⚠ DS E-Tense · rank 3050 · 1:23.456 [💢][📝]  [Edit]
```

- **Чекбокс**: автоматика — увімкнений, коли заповнений ефективний час.
  Автологіка діє, поки юзер не клацнув чекбокс руками; після ручного кліку
  ручний стан головніший, нові захоплення його не перетирають.
  `clear()` сесії скидає чекбокси назад в автоматичний режим.
- **Текст рядка**: ефективні значення. Перед непевним значенням — жовтий «⚠».
  Відсутні значення — як зараз: «трек?», «авто?», «ранг?», «час?».
  Якщо в драфті `bad_timing`/`note` — маленькі іконки (як у списку races:
  `icons/dislike.png`, `icons/info.png`; для note — тултіп з текстом).
- **Edit**: доступна для всіх 5 рядків, навіть без OCR-захоплення (можна вручну
  внести гонку, яку OCR пропустив). Відкриває `RaceDialog(relaxed=True)`,
  передзаповнений ефективними значеннями. `action` діалога пише результат у
  `session.set_draft(n, race_view)` — НЕ в БД.
- **Кнопки внизу**: `Save selected` + `Discard session`.
  `Save selected` активна ⟺ є хоч одна чекнута гонка І всі чекнуті повні
  (трек + авто + час). Неповні чекнуті гонки видно по «?» у рядку.
  `Discard session` активна, коли в сесії є хоч щось (OCR або драфт).

### 4. Збереження

- `Save selected`: для кожної чекнутої гонки будується `RaceView` з ефективних
  значень → `races_service.save(...)`.
- Помилка БД → `QMessageBox.critical`, сесія НЕ чиститься (можна виправити й повторити).
- Успіх → `challenge_session.clear()` (вся сесія, включно з нечекнутими гонками)
  + `QMessageBox.information` «N race(s) saved».

### 5. Оверлей

- `build_overlay_lines` переводиться з `dict[int, RaceCapture]` на
  `dict[int, EffectiveRace]` — оверлей і таб показують однакові дані.
- Позначка `✓`/`⚠` рахується по ефективних трек+авто+час.
- `CaptureController._refresh_overlay` бере дані через `session.effective(n)`.
- `bad_timing`/`note`/`rank` в оверлеї не показуються.

### 6. Видаляється

- `views/capture_review_dialog.py` повністю (фолбек рангу з БД авто вже
  покритий `RaceDialog.on_car_selected`).

### 7. Тести

- Юніт-тести на `effective()`: драфт перекриває OCR; порожнє поле драфта
  пропускає OCR; прапорці непевності (OCR-значення з низьким score → true,
  перекрите драфтом → false); `clear()` скидає драфти.
- Тест на `build_overlay_lines` з `EffectiveRace`.
- UI (`CaptureTab`) — ручна перевірка.
